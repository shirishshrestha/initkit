## Why These Changes?

### 1. **Avoid Config Conflicts**
Frameworks auto-generate their configs during `npx create-next-app`, `npm create vite`, etc. Our generated configs would conflict or override framework defaults.

### 2. **Respect Framework Conventions**
Each framework has its own recommended setup. Let the official CLIs handle initialization.

### 3. **Simpler Maintenance**
- Went from **600+ lines** per template to **~150 lines**
- No need to track config changes across framework versions
- Focus on folder structure, not implementation details

### 4. **Tailwind v4 Benefits**
- No config file needed (auto-generated)
- No PostCSS config needed
- No `@tailwind` directives in CSS
- Just install and use

## New Workflow

### For Users

**Before:**
```bash
initkit create my-app
cd my-app
npm install
# Configs might conflict with framework expectations
```

**Now:**
```bash
initkit create my-app  # Creates folder structure only
cd my-app
npx create-next-app . --use-npm  # Or npm create vite . -- --template react-ts
npm install  # Install additional libraries
npm run dev  # Start developing
```

1. **Organized folder structure**:

   - `src/features/dashboard/`

## Migration Guide

If you have existing projects using the old templates, here's how to update:

1. **Delete generated config files** (they'll be regenerated by framework)
2. **Run framework init command** in your project:
   - Next.js: `npx create-next-app@latest . --use-npm`
   - React: `npm create vite@latest . -- --template react-ts`
   - Vue: `npm create vue@latest . -- --typescript`
3. **Install Tailwind v4**: `npm install tailwindcss@next`
4. **Keep your feature folders** - they're compatible
